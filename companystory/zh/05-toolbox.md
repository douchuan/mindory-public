# 大厂那些事儿: 核心能力工具箱

当今的企业正面对:

- 增长放缓
- 经营压力加剧
- 预算与成本收紧

因此，用人企业遵循如下潜规则：

> **只筛选, 不培养**

如果你最近面过试，一定深有体会：

- 在线算法笔试
- 系统设计面对面
- 工具熟练度考察

这篇文章就来讲讲：你必须提前备好的 **入职工具箱**。

---

## 理论 + 实践

理论与实践, 两手都要抓，两手都要硬。

很多同学硕士毕业时，通常已经具备：
- 大量代码编写经验
- 科研或工程项目经历
- 扎实的问题解决能力

长期体系化训练，让他们的代码能力非常强。

但对本科生而言, 课业压力大且实战经验少，想本科毕业就直接参加工作, 最好从大一开始**系统性、刻意地刷题与做项目**。

具体来说：
- 坚持刷题
- 学习常用的开发框架
- 自己部署应用, 开发能上线的真实项目
- 尽早进厂实习

不要等到大四才 “临时抱佛脚”, 大厂面试，考的不只是知识，更是**熟练度**。 而熟练度，只能通过经年累月的**重复**才能习得。

---

## 知识积累

**终身学习**早已不是一句口号。

### 学习平台

- 全球顶尖技术演讲 ( 油管 )
- 系统化课程 ( B站, 极客时间 )

关键不在于“看过多少视频”, 而是，你必须**动手**。

---

## 写文档: 工程师的软实力

在大厂，写文档与做设计是工程能力的一部分。

### 文档能力

你需要做到：

- 写出结构清晰的技术方案
- 做出简洁实用的数据分析表
- 熟练使用 word

优秀的工程师，一定擅长表达与书写。

### 绘图与原型工具

- 思维导图工具 ( xmind )
- 产品原型工具 ( Axure )
- 流程图与架构图工具 ( Mermaid )

如果你无法清晰画出你的系统，多半说明，你还没有真正理解它。

---

## 核心技能

### 编程语言

要熟练掌握几门主流编程语言:

- Java / Go / Python
- TypeScript / JavaScript
- Shell 脚本

在 AI 时代, Python 是一门非常重要的编程语言，不会 Python，等于文盲。

### 开发框架

如果能熟练使用框架，进厂就能直接上项目。

- 前端开发框架 ( React, Vue )
- 后端开发框架 ( SpringBoot )

### 操作系统

你必须熟练使用：CentOS 和 Ubuntu, Linux 命令行操作是硬性要求。

### 网络调试工具

能理解网络行为，是普通开发者与优秀工程师的分水岭。

- 常用网络分析工具 ( ping, traceroute, netstat )
- 抓包工具 ( Wireshark, tcpdump )

---

## AI 工具：新时代的必备技能

AI 时代，生产力标准已被改写。 不使用 AI 工具, 已经不是这个时代的人了。

但请记住：AI 放大高手的能力，但无法替代底层基本功, 你有多强大，AI 就有多强大。

---

## 高效写代码：工具决定效率

### IDE 与编辑器

- 主流专业 IDE ( Visual Studio Code, IntelliJ IDEA )
- 轻量化高效编辑器 ( Sublime Text, Vim )

你不必精通 Vim，但需要具备基本的命令行编辑能力。

### 版本控制

你必须熟练使用 git 进行如下操作:

- 分支
- Rebase 与 Merge
- 提交 PR, 解决冲突
- 打 TAG

---

## 调试能力：高手的隐形门槛

熟练使用线上调试与诊断工具 (Alibaba Arthas, eBPF)，是资深工程师的标志。 谁都会写代码，但很少人能冷静定位并解决线上问题。

---

## DevOps 

今天的工程师，哪怕你不是运维, 也必须懂 DevOps:

- 容器 ( docker)
- 云服务与部署平台 ( Kubernetes )
- CI / CD 流程

---

## 企业内部工具：快速适应是硬要求

进入大厂后，你会接触大量内部系统：
- 自研依赖库
- 内部流水线
- 项目管理工具
- 缺陷追踪工具

没有人会从 0 教你, 企业默认你能**快速自学、独立上手**，一般会有入职应知应会手册，多看多问。

---

## 最后想说

现实很直白： 企业不再为“潜力”买单，他们只招聘“已准备好”的人。
