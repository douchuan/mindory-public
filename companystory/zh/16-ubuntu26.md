
#  Ubuntu 26.04 LTS 安全特性解读 

Canonical 在官方博客中明确表示，**Ubuntu 26.04 LTS 是他们“设计得最安全的 LTS 版本之一”**，通过同时提升系统每一层的默认安全基线（从固件到用户空间），
为桌面、服务器、虚拟机、云和边缘设备提供更强的长期防护。许多变化采用“默认启用 + 兼容回退”的策略，尽量减少对现有部署的破坏。

## Rust 工具 oxidising 策略

- sudo-rs 成为默认 sudo 实现，传统 sudo 作为后备保留
- Rust-based coreutils 取代 GNU coreutils 成为默认，GNU 版本仍可安装并通过 alternatives 系统切换

用 Rust 的内存安全特性替换长期以 C 语言实现的敏感工具，大幅降低缓冲区溢出等常见漏洞风险。这是 Canonical 在 25.10 中验证后，在 LTS 中正式落地的关键一步。

用户层面, 大多数命令行为保持一致，脚本兼容性较高。

## 基于 TPM 的全盘加密

这是本次 LTS 的重头戏，从 25.10 的实验功能升级为生产就绪：

- 新特性：安装后可在 Security Center 中轻松管理（添加/移除 PIN 或 passphrase、重新加密磁盘等）
- 官方重点改进：
  - 固件更新时恢复密钥处理更可预测（会在重启前明确提示）
  - 明确文档化已知不兼容情况（如 Absolute/Computrace）
  - 强化存储配置的内核模块要求。

优势：让硬件锚定的加密成为“一等公民”，减少更新时的意外，适合企业与移动设备部署。结合 Secure Boot，提供从固件到磁盘的完整信任链。

## Snap 权限提示

用户可见的 Snap 权限提示功能在 26.04 LTS 中仍处于实验阶段，并非默认启用。它代表了未来的发展方向，但尚未成为默认安全姿态。

这与部分早期信息不同，建议用户在正式发布后观察实际行为。若启用，将为 Snap 应用提供更细粒度的文件、设备、网络访问控制提示，提升透明度。

## 机密计算

- Intel TDX 和 AMD SEV-SNP 均获完整支持
- 允许运行内存被 CPU 加密且完整性保护的虚拟机，特别适合公共云、受监管行业和 AI workload 
- Ubuntu 在这方面的全栈集成（内核、固件、工具链）据称比其他发行版更完整

## 密码库 

- OpenSSH 升级至 10.2
- 默认启用混合后量子密钥交换：mlkem768x25519-sha256
- 完全移除 DSA 支持；SSH 服务器不再读取 `~/.pam_environment`，降低环境注入攻击风险。
- Web 服务器强化：Apache 2.4.66 默认禁用 TLS 1.0/1.1, Nginx 1.28.2 默认使用 TLS 1.2/1.3

这些变化无需手动开关，即可提供量子时代防护，同时保持向后兼容。

## AppArmor 与沙箱增强

- AppArmor 配置文件持续扩展，更多应用获得更严格的沙箱保护。
- Snap 权限提示（实验）进一步强化用户对 Snap 的控制。
- **注意**：新配置文件可能导致部分非标准使用场景出现问题，欢迎通过 Launchpad 反馈。

## 其他值得关注的安全强化

- **身份与目录服务权限降低**：
  - SSSD 以专用 `sssd` 用户而非 root 运行。
  - OpenLDAP 在 AppArmor enforce 模式下运行。
  - 引入 **authd** 作为支持的认证框架，可轻松集成云身份提供商（支持 OpenID Connect、MFA、条件访问）。

- **Secure Boot 与固件强化**：
  - 固件更新 + 更严格的 Secure Boot 策略。
  - NX（No-Execute）在所有变体中启用，移除旧的 strictnx 变体。
  - OVMF 固件针对 SEV/TDX 优化。

- **Security Center 成为安全控制平面**：
  - 安装后可在设置中直观查看和管理 TPM 加密状态、Secure Boot、磁盘保护等关键防护。
  - 将安全从“安装时一次性配置”转变为“全生命周期管理”。

- **Ubuntu Insights**：取代旧的 Ubuntu Report，完全 opt-in（需主动同意），旧同意不自动继承。提供图形化细粒度控制。
